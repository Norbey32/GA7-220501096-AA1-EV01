🔄 Versionamiento Local vs Remoto

Entiende las diferencias clave entre Git (local) y plataformas como GitHub (remoto).

🔧 Git (Local)

  • ✅ Almacenamiento: En tu computadora.
  • ✅ Acceso: Sin necesidad de internet.
  • ✅ Ejemplo: git commit guarda cambios localmente.

🔄 VS

🌍 GitHub/GitLab/Bitbucket (Remoto)

  • ✅ Almacenamiento: En servidores en la nube.
  • ✅ Acceso: Requiere internet.
  • ✅ Ejemplo: git push sube cambios al servidor.

📊 Tabla Comparativa

Característica Git (Local) GitHub/GitLab (Remoto)
Ubicación Tu máquina Servidores en la nube
Conexión No requiere internet Requiere internet
Comandos clave git init, git commit git clone, git push
Colaboración Limitada Multi-usuario

🚀 Flujo de Trabajo Típico

  1. Local: git add + git commit (guardas cambios).
  2. Remoto: git push (envías cambios al servidor).
  3. Equipo: git pull (otros descargan tus cambios).
# Ejemplo completo:
git add .
git commit -m "Fix bug"
git push origin main
